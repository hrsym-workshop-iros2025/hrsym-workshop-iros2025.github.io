Here is the information regarding the data names, descriptions, data types, and data formats for INT2 Complete scenario and split scenario.

Complete Scenario
Key Description Data Dimensions (each one) Data Type Value Range
DATA_ACQUISITION_TIME/begin The start time of data segment collection, specified precisely to the month, day, hour, minute, and second. {1} dict {time format}
DATA_ACQUISITION_TIME/end The end time of data segment collection, specified precisely to the month, day, hour, minute, and second. {1} dict {time format}
TIMESTAMP_SCENARIO The complete timestamp on the complete scene. [scenario_len, 1] float Time (s)
MAP_ID The hdmap ID corresponding to the scene [1] int32 {0~16}
SCENARIO_ID The name of the scene, named as "start time - end time". [1] string The unique name of the scene.
AGENT_INFO/object_id An integer ID for each object. [num_objects] int32 An object ID.
AGENT_INFO/object_type An integer type for each object (Vehicle, Pedestrian, or Cyclist) [num_objects] int32 BICYCLE=0, Pedestrian=1, Vehicle=2
AGENT_INFO/object_sub_type An integer type for each object (CYCLIST, MOTORCYCLIST, TRICYCLIST et al.) [num_objects] int32 CYCLIST=0, MOTORCYCLIST=1, TRICYCLIST=2, PEDESTRIAN=3, CAR=4, VAN=5, BUS=6, TRUCK=7
AGENT_INFO/state/position_x The x coordinate of each object at each time step. [num_objects, num_steps] float Global coordinates, meters.
AGENT_INFO/state/position_y The y coordinate of each object at each time step. [num_objects, num_steps] float Global coordinates, meters.
AGENT_INFO/state/position_z The z coordinate of each object at each time step. [num_objects, num_steps] float Global coordinates, meters.
AGENT_INFO/state/theta The theta coordinate of each object at each time step. [num_objects, num_steps] float Angle in radians. This is the angle to rotate a unit vector in the X direction in the X-Y plane to align with the length axis of the object bounding box.
AGENT_INFO/state/velocity_x The x component of the object velocity at each time step. [num_objects, num_steps] float m/s
AGENT_INFO/state/velocity_y The y component of the object velocity at each time step. [num_objects, num_steps] float m/s
AGENT_INFO/state/length The length of each object at each time step. [num_objects, num_steps] float Length in meters.
AGENT_INFO/state/width The width of each object at each time step. [num_objects, num_steps] float Width in meters.
AGENT_INFO/state/height The height of each object at each time step. [num_objects, num_steps] float Height in meters.
AGENT_INFO/state/valid A valid flag for all elements of features AGENT_INFO/state/XX. If set to 1, the element is populated with valid data, otherwise it is populated with -1. [num_objects, num_steps] int32 {0, 1}. Indicates if the data at each element of the AGENT_INFO/state/XX tensors is populated with valid data (rather than being padded with -1).
TRAFFIC_LIGHTS_INFO/tf_mapping_lane_id Road ID controlled by traffic signal lights. [num_traffic_lights] int32 An integer lane ID.
TRAFFIC_LIGHTS_INFO/tf_state_valid A valid flag for all elements of features TRAFFIC_LIGHTS_INFO/XX. If set to 1, the element is populated with valid data, otherwise it is populated with -1. [num_traffic_lights, num_steps] int32 {0, 1}
TRAFFIC_LIGHTS_INFO/tf_state The state of each traffic light at each time step. [num_traffic_lights, num_steps] int32 RED=0, GREEN=1, YELLOW=2
INTERACTION_INFO/interested_agents The ID of interested agents. [num_interested_agents] int32 An integer interested agents ID.
INTERACTION_INFO/interaction_pair_info/influencer_id The ID of influencer agent. [1] int32 An integer influencer agent ID.
INTERACTION_INFO/interaction_pair_info/reactor_id The ID of reactor agent. [1] int32 An integer reactor agent ID.
INTERACTION_INFO/interaction_pair_info/influencer_type The type of influencer agent. [1] int32 BICYCLE=0, Pedestrian=1, Vehicle=2
INTERACTION_INFO/interaction_pair_info/reactor_type The type of reactor agent. [1]] int32 BICYCLE=0, Pedestrian=1, Vehicle=2
INTERACTION_INFO/interaction_pair_info/coexistence_time The time when both influencer agent and reactor agent coexist. [num_coexit_step_indices] int32 A list of coexit step time's indices
INTERACTION_INFO/interaction_pair_info/interaction_time The index corresponding to the time during which there is interaction between influencer agent and reactor agent. [num_interaction_step_indices] int32 A list of interaction step time's indices
hdmap/LANE/has_traffic_control Whether the road is controlled by traffic signal lights. [1] bool {true, false}
hdmap/LANE/lane_type The type of road. [1] int32 BIKING=0, CITY_DRIVING=1, EMERGENCY_LANE=2, LEFT_TURN_WAITING_ZONE=3, ROUNDABOUT=4
hdmap/LANE/turn_direction Whether the road have a turning direction. [1] bool {true, false}
hdmap/LANE/is_intersection Whether it is an intersection. [1] bool {true, false}
hdmap/LANE/left_neighbor_id The ID of the adjacent lane on the left side. [1] int32 An integer left neighbor lane ID.
hdmap/LANE/right_neighbor_id The ID of the adjacent lane on the right side. [1] int32 An integer right neighbor lane ID.
hdmap/LANE/predecessors The lane ID that follows the current lane. [1] int32 An integer lane ID which follows current lane.
hdmap/LANE/successors The lane ID reached after crossing the current lane. [1] int32 An integer lane ID that will be reached after crossing the current lane.
hdmap/LANE/centerline "The centerline of the lane. [num_lane_points, 2] float A series of coordinate points representing the centerline of the road.
hdmap/LANE/left_boundary The left boundary of the lane. [num_lane_points, 2] float A series of coordinate points representing the left boundary of the road.
hdmap/LANE/right_boundary "The right boundary of the lane. [num_lane_points, 2] float A series of coordinate points representing the right boundary of the road.
hdmap/STOPLINE/centerline The stop line. [num_stoplane_points, 2] float A series of coordinate points representing the stop lane of the road.
hdmap/CROSSWALK/polygon The outer boundary line of the crosswalk. [num_crosswalk_points, 2] float A series of coordinate points representing the crosswalk of the road.
hdmap/JUNCTION/polygon The outer boundary line of the junction. [num_junction_points, 2] float A series of coordinate points representing the junction of the road.
hdmap/MAP_RANGE The extent of the intersection." [x_start, x_end, y_start, y_end] float The coordinate points defining the boundaries of the intersection.
Split Scenario

Split scenario is an extension of the complete scenario that includes an additional information configuration file.


Key Description Data Dimensions (each one) Data Type Value Range
split_interaction_info/interested_agents The ID of interested agents, just conclude influencer and reactor agent ID. [num_interested_agents] int32 An integer interested agents ID, , just conclude influencer and reactor agent.
split_interaction_info/split_time_91 The time range during which the interaction occurs. [2] int32 The time indices corresponding to the initial time and end time of the interaction occurrence.
